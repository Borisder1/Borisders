# Borisders

Наразі репозиторій знаходиться на дуже ранній стадії: у ньому є лише один файл з базовими git-командами для першого пушу.

## Що вже зроблено

- Додано `scripts/bootstrap_repo.sh` для безпечної ініціалізації репозиторію.
- Додано `PROJECT_REVIEW.md` з технічним аудитом та рекомендаціями розвитку.

## Швидкий старт

```bash
bash scripts/bootstrap_repo.sh
```

> Скрипт не перезапише ваш remote без підтвердження і перевіряє наявність `README.md`.

## Рекомендований план розвитку

1. Додати базову структуру проєкту (`src/`, `tests/`, `docs/`).
2. Налаштувати CI (мінімум lint + test).
3. Описати призначення проєкту та MVP у `docs/product.md`.
4. Додати issue template та PR template.

Детальний огляд див. у `PROJECT_REVIEW.md`.
